# 功能接口

## 功能一：单文件解析

**任务内容** 对单个媒体文件进行解析，提取描述的相关内容，并给出对应的内容解读。

**输入**：视频、音频、图片

**输出**：包括思考过程、事件、解读等

**输出示例**

```json
{
  "file": "us_drone_strike.mp4",
  "思考过程": "1. 逐帧分析：前8秒检测到无人机特征（旋翼声、热成像轮廓）；2. 第9秒检测到高速移动物体（导弹）与车辆交汇；3. 爆炸帧检测到火焰、碎片和冲击波光效；4. 音频分析：无人机嗡鸣(0-8秒)、导弹破空声(8.5秒)、爆炸声(9秒)、人群尖叫(9-15秒)；5. 综合判断：符合定点清除特征。",
  "事件": [
    "0-8秒：无人机在巴格达某街道居民区上空盘旋",
    "9-15秒：导弹击中一辆汽车，爆炸起火"
  ],
  "解读": "美军对疑似IRGC指挥官座驾实施定点清除，发生在居民区可能造成附带伤亡，预计将引发伊拉克议会反美抗议升级。"
}
```

## 功能二：多模态文件集合解析

**任务内容** 对一组多模态文件（视频、图片、音频、文本）分析，对每个场景融合信息并输出内容解读。

**输入**：多个文件（来自不同渠道，可能涉及不同事件）

**输出**：包括思考过程、事件、解读等

**输出示例**

```json
{
  "思考过程": "1. 引用任务三真伪结果：drone_video.mp4(真实)、witness_audio.wav(真实)、news_text.txt(真实)、fake_image.png(伪造)。2. 聚类：视频与音频在时间、地点、内容上匹配，归为同一事件；新闻文本内容一致但时区需校准；伪造图片排除。3. 冲突解决：视频显示白天，新闻称'夜间'，经核查时区差异后一致。",
  "场景分组": [
    {
      "场景ID": "scene_1",
      "包含文件": ["drone_video.mp4", "witness_audio.wav", "news_text.txt"],
      "事件": [
        "美军无人机在巴格达机场附近击中一辆SUV",
        "导致至少3人死亡，含一名伊斯兰革命卫队高级指挥官"
      ],
      "解读": "此次空袭是'斩首行动'，将激化伊朗代理人武装报复。未来72小时，伊拉克境内美军基地遭火箭弹袭击概率极高。"
    }
  ]
}
```

## 功能三：媒体文件真伪鉴别

**任务内容** 对单个媒体文件（视频/图片/音频）进行真伪鉴别，输出思考过程、结论等

**输入**：单个媒体文件

**输出**：包括思考过程、结论、置信度、可疑点等

**输出示例**

```json
{
  "file": "fake_video.mp4",
  "思考过程": "1. 视觉：人脸边缘锯齿状伪影，眼睛高光方向不一致；2. 时序：帧间光流在人物轮廓处断裂；3. 音频：口型延迟0.2秒，且背景电频噪声在1.5秒处跳跃。",
  "结论": "伪造",
  "置信度": 0.94,
  "可疑点": ["人脸边缘伪影", "光流断裂", "口音不同步", "背景噪声跳跃"]
}
```

```json
{
  "file": "real_protest_video.mp4",
  "思考过程": "1. 频域分析：未检测到GAN生成特征，频谱分布自然；2. 光流法：人群运动轨迹连续，无明显断裂或扭曲；3. 口唇同步：语音与口型延迟<0.05秒，属拍摄正常范围；4. 元数据：编码参数一致，无异常时间戳；5. 压缩痕迹：整体JPEG块均匀，无局部重压缩区域。",
  "结论": "真实",
  "置信度": 0.96,
  "可疑点": []
}
```
